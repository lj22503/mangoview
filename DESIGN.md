# MangoView Design System v2.0

> **方向**: 温暖金融品牌 — 有知有行 × 富途
> **原则**: 让每个投资者都能建立自己的体系
> **日期**: 2026-06-15
> **状态**: 重构中

---

## 0. 叙事哲学（Narrative Philosophy）

### 核心信念

**投资的差别不在信息，在于是否拥有完整的体系。**

MangoView 不贩卖焦虑，不制造 FOMO。我们做的事很简单：把经典投资框架翻译成普通人能用的工具。自上而下不是一句口号，是每一步都有依据、每一个决策都可追溯。

### 叙事原则

| 原则 | 做什么 | 不做什么 |
|------|--------|----------|
| **体系优先** | 引导用户建立自己的框架 | 不推个股、不喊单 |
| **说人话** | 用类比解释复杂概念 | 不堆术语制造专业幻觉 |
| **给依据** | 每个结论标注数据来源和逻辑链 | 不凭空给"强烈推荐" |
| **陪伴感** | "我们一起来看"而非"你应该" | 不说教、不居高临下 |
| **承认局限** | 明确标注数据盲区和不确定性 | 不假装全知全能 |

### 类比质量红线

类比是为了降低理解门槛，不是为了凑热闹。每条类比必须满足：

- **对应关系清晰**: "A 就像 B，因为 C" 三要素齐全
- **不误导**: 类比不能扭曲原概念的核心逻辑
- **不碎嘴**: 一个概念最多一个核心类比，不堆叠

✅ 好类比：
> 基钦周期就像小卖部进货——货多了降价清仓，货少了涨价补货，三四年一个来回。这就是"库存的小波浪"。

❌ 坏类比：
> 基钦周期就像吃饭睡觉打豆豆，三四年来一次。

### 页面叙事结构

每个数据页面遵循三层叙事：

1. **这一页看什么**（一句话定位）
2. **怎么看**（类比重述 + 指标解读）
3. **现在是什么情况**（数据呈现 + 当前位置标注）

## 1. Visual Theme & Atmosphere

### 设计哲学

**"温暖的精确"**

专业投资工具不需要冷冰冰。MangoView 的视觉语言是：暖橙色基底承载专业数据，清晰卡片结构让信息触手可及。像一杯温度刚好的芒果茶 — 不烫嘴，不冰牙，刚刚好能品出味道。

### 情绪板

| 维度 | 描述 |
|------|------|
| **情绪** | 温暖、可信赖、有条理 |
| **密度** | 适中偏密 — 数据有呼吸感但不空荡 |
| **节奏** | 稳定的卡片流，重点数据用暖色打破 |
| **温度** | 暖而不燥 — Mango 橙主导但大面积中性色平衡 |
| **风格** | 现代金融品牌 + 投教内容感 |

### 参考产品

- **有知有行**: 叙事方式 — 说人话的体系化投教
- **富途牛牛**: UI 密度与结构 — 数据清晰、模块分明
- **Robinhood**: 品牌温度 — 让金融不那么吓人
- **Koyfin**: 数据可视化 — 图表即答案

### 颜色使用规则

- **橙色层级**: Mango 100 → 200 → 400 → 500 → 600 构成完整暖色梯度
- **红涨绿跌**: 严格遵循中国股市习惯（红色上涨、绿色下跌）
- **橙色克制**: 仅用于品牌标识、CTA、核心数据高亮，一屏不超过 3 处
- **大面积暖色**: 仅限 Surface Warm / Mango 100 做区块底色，不滥用
- **蓝色仅用于**: 链接、信息提示，不用于金融数据

## 8. Do's and Don'ts（更新）

### ✅ Do's

- 使用 Mango 暖橙体系的三层梯度（100/200/400/500/600）
- 数据表格用等宽字体 JetBrains Mono
- 卡片 14px 圆角 + 1px 暖灰边框 + E1 阴影
- 红涨绿跌，严格遵循中国习惯
- 每个数据页面三层叙事（看什么 → 怎么看 → 现在怎样）
- 类比必须满足对应关系清晰、不误导、不碎嘴
- 亮色默认，暗色通过 toggle 切换
- 用 Lucide 图标，禁止 emoji 做 UI 元素

### ❌ Don'ts

- 不要用 emoji 做功能图标
- 不要在数据表格中用斜体
- 不要用蓝色表示金融数据涨跌
- 不要用超过一处的大面积纯橙色
- 不要说教式文案（"你应该""你必须""聪明人都"）
- 不要制造信息焦虑（"错过这个就晚了"类话术）
- 不要用渐变背景（暖色通过区块底色实现，不用 gradient）
- 不要把扁平化用在卡片上（必须有 1px 边框 + 阴影）
