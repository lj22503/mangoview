## 一、产品对客逻辑

### 1.1 用户是谁

**目标用户**：知识星球订阅者（不是开发者自用）

**核心痛点**：
- 市场怎么看？
- 该不该买？
- 该买啥？

**产品价值**：自上而下系统化判断，比凭感觉准

### 1.2 用户旅程（4 场景，对客视角）

| 步骤 | 场景 | 入口 | 产出 |
|------|------|------|------|
| 1 | 订阅后首次访问 | Landing 页 | 知道产品是什么 |
| 2 | 每天被动接收 | 飞书日报推送 | 5 分钟知道今日信号 |
| 3 | 主动登录使用 | mangoview 仪表盘 | 市场看版 + 工具集 + 报告页 |
| 4 | 复盘 | 历史报告 | 验证历史判断的准确性 |

### 1.3 信息流自洽性

```
数据层 → 信号层 → 决策层 → 报告层 → 飞书推送
13 指标  5 因子   工具集  日报/周报 ↓
行业数据 周期位置  配置建议          [天则·识局独立消费]
```

**关键约束**：报告 ≠ 独立内容，是仪表盘核心数据的不同频率摘要

### 1.4 日报 vs 周报（内容差异）

| | 日报 | 周报 |
|--|-----|-----|
| 关注重心 | 当日新增 / 信号阈值 / 警示事件 | 周度趋势 / 配置建议 |
| 数据类型 | 日级指标变化 | 周度汇总 + 跨周期对比 |
| 长度 | 5 分钟读完 | 15 分钟读完 |
| 产出渠道 | 飞书 + 报告页 | 飞书 + 报告页 |

## 二、整合范围

| 产品 | 角色 | 关键约束 |
|------|------|---------|
| **mangoview** | 对客产品（核心，mangoview 2.0） | 唯一对客入口 |
| **宏观仪表盘** | 数据采集 + 信号引擎 + **临时独立 UI** | UI 临时保留防止改错，mangoview 稳定后再优化 |
| **周期仪表盘** | 战略层参考文档 | 13 指标定义 + 5 维度诊断，挑能用的进 mangoview |
| **天则·识局** | 独立 Skill | **不在整合范围**，消费统一数据 API |

## 三、存储方案（0 服务原则）

- **用户配置 / Portfolio**：浏览器 localStorage
- **报告历史 / 数据快照**：GitHub Raw JSON（按日期 commit）
- **采集数据**：本地 daily_collector + GitHub Actions

**不引入**：Turso / Supabase / Neon 等数据库服务

---

## 四、部署架构（Vercel 一站式）

```
┌─────────────────────────────────────────┐
│            Vercel (单一账号)            │
├─────────────────────────────────────────┤
│  mangoview 前端 (Next.js 14)            │
│  宏观仪表盘前端 (Vite + React)          │
│  FastAPI 后端 (Mangum 适配)             │
└─────────────────────────────────────────┘
              ↑ HTTPS
              │
┌─────────────┴───────────────────────────┐
│        数据采集层（不上云）              │
├─────────────────────────────────────────┤
│  本地 daily_collector.py (实时/小时)    │
│  GitHub Actions (日级 08:30 / 14:30)    │
│  akshare / FRED / yfinance              │
└─────────────────────────────────────────┘
              ↑ commit JSON
              │
┌─────────────┴───────────────────────────┐
│       GitHub Raw JSON (数据仓库)         │
├─────────────────────────────────────────┤
│  data/all_indicators.json               │
│  data/archives/YYYY-MM-DD.json          │
│  data/reports/YYYY-MM-DD.json           │
└─────────────────────────────────────────┘
```

**关键决策**：
- ❌ Render（绑卡）
- ❌ 阿里云 PM2（IP 被东财限制）
- ✅ Vercel Serverless（免费、无绑卡、统一管）

## 五、数据采集频率分级

| 频率 | 数据类型 | 采集位置 | 落地方式 |
|------|---------|---------|---------|
| **小时/实时** | 北向资金、行情 | 本地 daily_collector | 走 API 直读 |
| **日级** | PMI、CPI、社融、FRED 资产价格 | GitHub Actions (08:30/14:30) | commit JSON |
| **周/月级** | 货币金融数据、社融存量 | 月底/周末触发 | commit JSON |

## 七、风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| Vercel Serverless 10s 超时 | 长任务失败 | 数据采集本地，不进 Serverless |
| Vercel 100GB-h/月带宽 | 高流量被限流 | 静态资源走 CDN，API 走 Serverless |
| 阿里云 IP 仍限制 | 部分数据无 | 采集全部本地，GitHub Actions 中转 |
| localStorage 跨设备不同步 | 体验割裂 | 第一阶段接受，后续可加 Turso |
| 两个 UI 同时对外 | 对客入口混乱 | 明确主入口是 mangoview，宏观仪表盘标记为 legacy |
